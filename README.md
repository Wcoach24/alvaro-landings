# Alvaro Landings

Per-role landing pages generated by **JobHunter v2** (Track A — bold) for senior AI roles at frontier labs / AI scaleups (score ≥ 9). Each landing adopts the host company's visual identity (Anthropic warm cream + Lora · Vercel pure black + Geist · Linear dark purple · Mistral bold European · etc.).

## Live

- Production: https://for.alvarozamorano.com
- Vercel: alvaro-landings.vercel.app

## How a landing gets built

```
1. JobHunter v2 scoring detects a Tier-A role at score ≥ 9 (frontier lab / AI scaleup)
2. scripts/generate_landing.py --brief writes data/{slug}.brief.md
3. The Cowork agent (already inside Claude) reads the brief and writes data/{slug}.json
4. git push → Vercel auto-deploys in ~60s
5. URL goes in the cold email subject: "Built this 47min after seeing your JD: alvaro.tools/{slug}"
```

## Structure

```
landings/
├── app/[slug]/page.tsx        ← dynamic per-slug landing
├── components/                 ← HeroLive, JDMatch, LiveAgents, CTA, Disclosure
├── lib/
│   ├── brandProfiles.ts        ← per-company visual identity presets
│   ├── brandStyle.ts           ← CSS generator from a BrandProfile
│   ├── types.ts                ← LandingData shape
│   └── loadData.ts             ← fs read of data/{slug}.json at build time
├── data/{slug}.json            ← one JSON per landing (committed)
└── public/cv/cv_en.pdf         ← Álvaro's CV
```

## Adding a new company brand

Edit `lib/brandProfiles.ts` and add a `BrandProfile`. Inspect the company's real site (CSS vars, theme-color, font stacks) — don't invent colors.

## Disclosure stance

Every landing has a footer making automation explicit. Filter mechanism: companies that appreciate technical capability respond; others self-select out. Used **only for Track A** (frontier labs / AI scaleups), never for recruiter agencies.
